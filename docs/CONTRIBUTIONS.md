# Team Contributions

Development was initially performed in separate repositories, with each team member implementing their own version of the project while following a shared architecture and interface design.

Once a stable baseline had been established, the team migrated to a synchronized repository. Components from the individual implementations were merged into a common codebase, after which development became highly collaborative.

## Yves Mangano

* Developed the initial blockchain data structures.
* Implemented early versions of the block, block header, and transaction classes.
* Implemented blockchain utility functions used throughout the project.
* Contributed to integration and refinement of the blockchain layer.

## Jayran Duggins

* Refactored registration functionality into a dedicated registration module and overlay.
* Implemented significant portions of the blockchain management logic.
* Contributed to blockchain networking and integration.
* Assisted with protocol design and system architecture.

## Darian Samsoedien

* Implemented the mining subsystem and block production workflow.
* Contributed to blockchain networking and payload design.
* Assisted with synchronization and integration of blockchain components.

## Shared Contributions

The following components were developed collaboratively:

* Blockchain community design
* Network message formats
* Message handlers
* Mempool implementation
* Block validation
* Chain synchronization
* Integration testing
* Debugging and stabilization

Most architectural decisions were discussed collectively and implemented through collaborative development sessions.
